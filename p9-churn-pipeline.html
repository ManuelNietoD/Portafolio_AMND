El Problema de Negocio
Un banco pierde 450,000 clientes al año.
La retención reactiva llega demasiado tarde.
Un banco retail mexicano con 2.5 millones de clientes enfrenta una tasa de abandono del 18% anual — equivalente a perder $3,780 millones de MXN en ingresos futuros cada año. El problema no es que los clientes se vayan: es que el banco solo lo sabe después de que sucede.

La retención reactiva (llamar al cliente cuando ya pidió cancelar) tiene una tasa de éxito del 12%. La retención preventiva — contactar al cliente antes de que decida irse — tiene una tasa de éxito del 42%. La diferencia es predecir el churn con suficiente anticipación. Eso es exactamente lo que resuelve este modelo.

M2 Limpieza & Preprocesamiento Missing 3.2% · Outliers 0.8% · Encoding ▾
Decisiones de Preprocesamiento
Missing values (3.2%): imputación por mediana para variables numéricas (ingreso reportado, saldo mínimo). Categoría "Desconocido" para variables categóricas (tipo de ocupación).

Outliers (0.8%): Winsorización al percentil 99 para ingresos y saldos — clientes ultra-HNW no se eliminan pero se capean para no distorsionar el modelo.

Encoding: Target encoding para variables categóricas de alta cardinalidad (municipio, segmento NPS). One-hot para variables binarias (canal preferido, tipo de cuenta).
Missing imputados: 80K Outliers tratados: 20K Vars finales: 68

M3 Feature Engineering 28 features construidas · RFM · Comportamentales ▾
Features Construidas por Categoría
RFM (Recency-Frequency-Monetary): días desde última transacción, frecuencia mensual promedio, monto promedio por transacción. Tendencia de los últimos 3 vs. 6 meses.

Ejemplo: Feature de Velocidad de Deterioro
# Feature: tendencia de saldo últimos 90d
def calc_balance_trend(df):
    df['saldo_trend_90d'] = (
        df['saldo_promedio_t0'] -
        df['saldo_promedio_t90']
    ) / df['saldo_promedio_t90'].clip(1)

    # Velocidad: % de caída por semana
    df['velocidad_deterioro'] = (
        df['saldo_trend_90d'] / 13
    )
    return df

M4 Modelado · 5 Algoritmos Comparados LightGBM · XGBoost · RF · Logístico · Red Neuronal ▾
Estrategia de Modelado
Desbalance de clases: SMOTE en train (ratio 1:3), class_weight='balanced' como alternativa. SMOTE superó class_weight en +1.8pp F1.

Validación temporal: train en meses 1–18, validación en meses 19–21, test en meses 22–24. No shuffle aleatorio — se respeta la secuencia temporal para evitar data leakage.

Hyperparameter tuning: Optuna con 100 trials para LightGBM y XGBoost. Early stopping con 50 rondas en validación.

Calibración: Platt scaling post-training para que las probabilidades sean interpretables como probabilidades reales de churn.

Resumen Ejecutivo
El banco enfrenta una tasa de abandono del 18% anual (450,000 clientes), equivalente a $3,780 millones MXN en ingresos futuros perdidos. Este proyecto desarrolló un modelo de machine learning (LightGBM) que predice con 90 días de anticipación qué clientes abandonarán el banco, con un AUC de .924 validado en datos out-of-time.

La implementación de retención preventiva sobre el top-20% de clientes en riesgo generaría un ROI neto de $294M MXN en el primer año, con payback del proyecto en 3 meses. El modelo está en producción desde el Q2 2024 y procesa el portafolio completo de 2.5M clientes diariamente.
